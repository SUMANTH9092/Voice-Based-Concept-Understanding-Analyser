# Software Requirements Specification (SRS) - Voice-Based Concept Understanding Analyser

This document describes the functional, non-functional, and user requirements for the VBCUA application.

---

## 1. Project Scope & Definitions

### Scope
VBCUA is a self-contained local web application built with Streamlit. It accepts audio recordings of students explaining a concept, transcribes the audio using OpenAI Whisper, evaluates semantic accuracy using Sentence-BERT, analyses vocal patterns using Librosa, and generates a comprehensive PDF assessment report.

### Definitions
- **Reference Concept**: The gold-standard textbook definition of a concept provided by the user or educator for comparison.
- **Transcript**: The text output generated by OpenAI Whisper from a student's audio recording.
- **Cosine Similarity Score**: A numerical value between 0 and 1 indicating the semantic similarity between the transcript and the reference concept.
- **Understanding Grade**: A categorical label (Excellent / Good / Needs Improvement / Poor) derived from the cosine similarity score.
- **Filler Words**: Disfluency markers in speech (e.g., "um", "uh", "like", "you know", "basically") that indicate hesitation or incomplete thought.
- **RMS Energy**: Root Mean Square energy of the audio signal, used as a proxy for vocal projection and confidence.

## 3. Functional Requirements

### FR-01: Audio File Upload
- The system must accept audio file uploads in `.wav`, `.mp3`, `.m4a`, and `.ogg` formats.
- The system must reject audio files exceeding the configured `MAX_AUDIO_DURATION` (default: 300 seconds).
- The system must display an error message for unsupported file types.

### FR-02: Speech Transcription
- The system must transcribe the uploaded audio using OpenAI Whisper.
- The system must support selection of Whisper model size (`tiny`, `base`, `small`, `medium`, `large`) via configuration.
- The system must display the generated transcript in the UI before analysis.

### FR-03: Semantic Similarity Analysis
- The system must encode both the transcript and the reference concept using Sentence-BERT (`all-MiniLM-L6-v2`).
- The system must compute cosine similarity between the two embeddings and display the score (0.00 – 1.00).
- The system must assign a categorical understanding grade based on the similarity score:
  - **Excellent**: score ≥ 0.90
  - **Good**: 0.75 ≤ score < 0.90
  - **Needs Improvement**: 0.50 ≤ score < 0.75
  - **Poor**: score < 0.50

### FR-04: Vocal Analytics
- The system must detect and count filler words in the transcript.
- The system must use Librosa to detect silence segments (pauses) and report the total count and average duration.
- The system must compute the RMS energy of the audio signal and display it as a vocal confidence indicator.

### FR-05: Waveform Visualization
- The system must render an amplitude waveform plot of the uploaded audio using Matplotlib/Librosa Display.
- The system must optionally display a Mel spectrogram for advanced analysis.

### FR-06: PDF Report Generation
- The system must generate a downloadable PDF report containing:
  - Student name (if provided), date/time, concept name.
  - Full transcript text.
  - Semantic similarity score and understanding grade.
  - Filler word count and list.
  - Pause count and average duration.
  - RMS energy value.
  - Embedded waveform plot image.

## 4. Non-Functional Requirements

### NFR-01: Performance
- Transcription of a 60-second audio clip using the `base` Whisper model must complete within 30 seconds on standard consumer hardware (Intel i5/i7 with 8GB RAM).
- The Streamlit dashboard must load and render all results within 5 seconds of processing completion.

### NFR-02: Usability
- The UI must be operable without any prior technical training — designed for students and educators, not engineers.
- All results must be presented with clear labels and explanatory tooltips.

### NFR-03: Reliability
- The system must handle corrupt or silent audio files without crashing, returning a descriptive error message.
- All exceptions must be caught and logged to stdout rather than exposed as raw stack traces in the UI.

### NFR-04: Portability
- The application must run on Windows, macOS, and Linux without platform-specific configuration.
- All dependencies must be installable via `pip install -r requirements.txt`.
